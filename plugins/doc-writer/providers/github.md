# Provider: GitHub

Use this provider when `git remote get-url origin` contains `github.com`.

## How this fits with the rest of the plugin

- **Reading / analysis** — Use **git** for diffs, logs, and file contents: `git diff`, `git log`, `git show HEAD:<file>`. No `gh` needed for this.
- **GitHub-specific** — Use **`gh`** only to resolve the PR number when not passed as an argument, post comments, and open follow-up PRs.

## Prerequisites for posting

- **GitHub CLI** (`gh`) installed: https://cli.github.com
- Authenticated: `gh auth login`, or non-interactive via `GH_TOKEN` / `GITHUB_TOKEN`
- **Token scopes:** `repo` (private repos) or `public_repo` (public); `Contents: Read & Write`, `Pull requests: Read & Write`

---

## Resolve the PR Number

If a PR number was passed as an argument, use it directly.

Otherwise, for the current branch:

```bash
gh pr view --json number --jq '.number'
```

Or:

```bash
gh pr list --head "$(git rev-parse --abbrev-ref HEAD)" --json number --jq '.[0].number'
```

Parse `owner` and `repo` when needed:

```bash
REMOTE=$(git remote get-url origin)
OWNER=$(echo "$REMOTE" | sed 's|https://github.com/||;s|git@github.com:||' | cut -d'/' -f1)
REPO=$(echo "$REMOTE"  | sed 's|https://github.com/||;s|git@github.com:||' | cut -d'/' -f2 | sed 's|\.git$||')
```

---

## Posting the "Documentation Update in Progress" Comment

```bash
gh pr comment <pr-number> --body "$(cat <<'EOF'
📝 **Documentation update in progress**

I'm inspecting the code changes in this PR and updating any relevant documentation. The results will be committed to this branch (or a follow-up PR if already merged) when complete — this may take a few minutes.
EOF
)"
```

If posting fails, output one warning line and continue.

---

## Posting the Summary Comment (Pre-merge)

After committing updated documentation directly to the PR branch, post a summary comment:

```bash
gh pr comment <pr-number> --body "$(cat <<'EOF'
📝 **Documentation updated**

The following documentation files were updated to reflect the code changes in this PR:

${SUMMARY_BODY}

Changes have been committed directly to this branch.
EOF
)"
```

---

## Opening a Follow-Up PR (Post-merge)

When the original PR has already been merged, push the documentation changes to a new branch and open a follow-up PR:

```bash
gh pr create \
  --title "docs: update documentation for merged changes from #<original-pr-number>" \
  --body "$(cat <<'EOF'
## Summary

This PR updates documentation to reflect the code changes that were merged in #<original-pr-number>.

### Files changed

${SUMMARY_BODY}

🤖 Generated by the doc-writer plugin.
EOF
  )" \
  --base <base-branch> \
  --head <docs-branch>
```

---

## Output

On completion:

```
Documentation updated: <N> files modified, <N> files created — committed to <branch>
```

Or for post-merge:

```
Documentation updated: <N> files modified, <N> files created — follow-up PR: https://github.com/<owner>/<repo>/pull/<number>
```
